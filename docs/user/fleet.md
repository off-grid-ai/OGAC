# Fleet

*Skeleton (how/what/why/when) — to be deepened.* Surface: **Gateway & Fleet → Fleet (`/fleet`)**.

## What it is

Devices, enrollment, policy assignment, and the kill switch for the device fleet.

## Why use it

- Know and control which devices are enrolled.
- Assign policy; kill a compromised device instantly.

## When to use it

- Onboarding/offboarding a device.
- Responding to a lost/compromised device (kill switch).

## How to use it

Enroll a device, assign a policy, view status, and trigger the kill switch. See `docs/FLEET_CONTROL.md` for the control model.
