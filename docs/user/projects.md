# Projects

*Skeleton (how/what/why/when) — to be deepened.* Surface: **Workspace → Projects (`/projects`, reached from Chat's project switcher + Workspace tabs)**.

## What it is

Group chats under shared instructions and a dedicated knowledgebase — a workspace per topic (ChatGPT/Claude Projects parity). A project is a chat context, not a separate app.

## Why use it

- Keep a topic's chats, instructions, and reference docs together.
- Give every chat in a project the same system instructions and grounding automatically.

## When to use it

- Standing bodies of work (a team, a case, a product) that recur across many chats.
- When you want consistent instructions + knowledge applied without re-attaching each time.

## How to use it

Create a project (name, shared instructions, upload/attach its knowledgebase). Start chats inside it from Chat's project switcher. Edit instructions or the knowledgebase anytime; delete a project when it's done. Reached via Chat's switcher and the Workspace top-tabs (not the sidebar).
