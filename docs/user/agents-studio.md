# Agents & Studio

*Skeleton (how/what/why/when) — to be deepened.* Surface: **Intelligence → Agents (`/agents`) and Studio (`/studio`)**.

## What it is

**Agents** is a catalog of pre-built AI agent use cases. **Studio** is where you build agents & workflows in plain language — wired to your connectors, data, tools, and guardrails.

## Why use it

- Ship value fast with pre-built agents; then author your own without code.
- Every agent runs through the governed pipeline and is recorded (see Agent Runs).

## When to use it

- Use Agents to adopt a ready use case.
- Use Studio to build a bespoke assistant/workflow over your data + tools.

## How to use it

Browse and run a pre-built agent from **Agents**. In **Studio**, describe the agent/workflow in plain language, wire it to connectors/data/tools/guardrails, test it, and save it. Runs land in [Agent Runs & Jobs](agent-runs-jobs.md); create/edit/delete agents from these surfaces.
