Off Grid AI — the platform

Become an intelligent enterprise, without compromising. On-prem · local-first · open-source · data never leaves the box.

One governed pipeline is the unit of everything — it governs model access AND data movement under one policy/guardrail/redaction/eval contract. It runs on a smart gateway (routing + edge + its own telemetry). Every run and every sync emits into one shared spine (observability · audit · lineage · provenance · cost · compliance) — replayable end-to-end — and an org view rolls the whole estate up. That's the platform.

Consumers — who uses a pipeline (Apps vs Agents is the key distinction)

APP — has a UI
Human-facing. A UI + input forms · HITL (human-in-the-loop approve/reject/edit) · a Review queue · Reports. A person drives it. e.g. reimbursement approval, KYC review.
AGENT — no UI
Autonomous. No UI, runs on a trigger/schedule, does its work headless, and emits a report. No human in the loop. e.g. overnight fraud sweep, scheduled data sync, monitoring.

Pipeline — the one governed unit (owns everything)

What it can DO
Model access → grounded answers + citations.  Data movement → connector→warehouse sync (batch · incremental · CDC).  Compose pipelines-as-tools.
How it's consumed
provisioned API key + endpoint · versioned + replayable (every run/sync is a recorded, re-runnable event) · templated (sample BFSI pipelines)
Governed by the SAME peripheries whether it's moving tokens or rows:
data-allowlist — hard ceiling policy (ABAC / OPA) guardrails (injection · grounding · toxicity) PII redaction / masking (Presidio) — on the sync path too model evals + golden set drift detection data-quality evals (Great Expectations) routing + egress leash schedule / CDC trigger RBAC (who may run) → ABAC (what this request may touch)
Per-pipeline lenses = its slice of the spine: observability · audit · lineage · provenance · cost · quality.

The experience bet — beautiful & effortless

Creating and deploying a governed use-case is the core loop, and it must be a joy, not a config chore. You start from a prebuilt pipeline (governance already wired), describe the change in plain English, and deploy — as an App (UI + HITL + review + reports) or an Agent (autonomous, report-only). The library of small composable pipelines is the on-ramp; the plain-language authoring is the magic; the shared spine means every use-case is observable, auditable, and replayable the moment it ships.

Positioning: not "a private ChatGPT" — a private AI, everywhere, and the governed data platform under it. Open-core: free under 20 people; 20+ license from us. The bet: every enterprise capability (model access, ETL, BI, governance, compliance) as small, reusable, composable, replayable pipelines — shipped as fully-governed examples — that never let data leave.
